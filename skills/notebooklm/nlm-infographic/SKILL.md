# nlm-infographic — Professional NotebookLM Infographic

Build a source-grounded infographic via the NotebookLM MCP. Best as one "at-a-glance" summary slide or a printable leave-behind.

## Capability ceiling (lower than slides — say so)
Generated by an image model (Nano Banana Pro), so control is **probabilistic, not deterministic** like the slide-layout engine:
- **Reliable:** content focus/emphasis, which stats to feature, **palette (incl. hex)**, **named style**, **orientation**, density via `detail_level`.
- **Loose / not guaranteed:** exact section count + order. A spec biases it strongly but the model interprets.
- **Unreliable — verify every time:** exact label/number **text** (typos persist even when the value is verbatim in the source, even if you tell it to check spelling).
- **No revise. PNG-only, not editable in-platform.** Iteration = **regenerate**; fix a single wrong label in an **external OCR editor**. Free tier ~3 generations/day; AI-inaccuracy disclaimer applies.

Keep it to **3–5 sections** — more degrades fidelity and triggers typos.

---

## Step 0 — Auth check
`mcp__notebooklm__server_info`; if not `configured`, have the user run `! nlm login`.

## Step 0b — Reuse a prepped notebook?
If the user ran `/nlm-source-prep` (or has a curated notebook), ask for the `notebook_id` and reuse it — skip source-gathering and ingestion in Steps 1 + 4, and go to Step 2 (presets).

## Step 1 — Ask for sources (unless a prepped notebook was supplied)
**3–5 tightly-scoped sources** (one notebook = one artifact). Files / URLs / Drive / text via matching `source_add` types. Ask what the **single key takeaway or the 3 findings** to visualize are — infographics work best with a tight focus.

## Step 2 — Set professional presets
Defaults for a work context:
- `infographic_style`: **`professional`** (others: `auto_select`, `editorial`, `bento_grid`, `instructional`, `scientific`, plus stylized ones — style customization needs 18+).
- `orientation`: **`landscape`** (fits a 16:9 slide) | `portrait` | `square`.
- `detail_level`: **`standard`** | `concise` | `detailed`.
- `language`: BCP-47, default `en`.

## Step 3 — Ask: spec the layout, or let NotebookLM decide? (same fork as /nlm-deck)
Use AskUserQuestion:

- **"I'll spec the sections"** *(recommended for control)* — the user lists the blocks/metrics (or Claude drafts ≤5 from the verified claims for approval). Write that as a **text source** titled `INFOGRAPHIC_SPEC` via `source_add(source_type="text", title="INFOGRAPHIC_SPEC", text=<spec>, wait=true)`, pin `source_ids` to it, and restate the order in the focus prompt. Honest caveat: this **biases strongly but does not lock** section order, and won't fix label typos.
- **"Let NotebookLM decide"** — supply only the key findings + emphasis in the focus prompt.

## Step 3b — Build focus_prompt
Restate structure explicitly, name the hero stat, give an exact palette, request contrast. e.g.:
> "Single-page infographic. Lead with the hero stat <X>, then <N> supporting sections in this order: <…>. Professional style, palette <hex / 'corporate blue + dark gray'>, maintain WCAG AA contrast. Emphasize the <before/after metric>. No decorative filler."

If specced, add: "Base the content strictly on the source titled INFOGRAPHIC_SPEC."

## Step 4 — Ingest + generate
`notebook_create` → `source_add(... wait=true)` → confirm → `studio_create(notebook_id, artifact_type="infographic", infographic_style="professional", orientation="landscape", detail_level="standard", focus_prompt=..., source_ids=<relevant>, language=<bcp47>, confirm=true)`.

## Step 5 — Poll + deliver
Loop `studio_status(notebook_id)` until ready, then `download_artifact(notebook_id, artifact_type="infographic", output_path=...)` → PNG.

## Step 6 — Iterate + QA (no revise — plan for this)
- **Verify every label and number against its source** — text errors are the #1 infographic failure and persist even when the source is correct.
- **To change anything: regenerate**, don't expect an edit. Tighten the focus prompt, cut sections, restate the exact label, and keep `detail_level` at `standard`/`concise` (never `detailed` — it's the main typo trigger).
- **One wrong label?** Route it to an external **OCR editor** (e.g. PDNob) to fix text on the PNG while keeping the layout — re-prompting the same image tends to make it worse. Canva can only overlay, not edit baked-in text.
